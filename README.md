# Дневник вайб-кодинга

## Описание

Личный дневник заметок о вайб-кодинге: сессии с ИИ, инсайты, инструменты и шаги на пути от новичка к автоматизациям, маркетингу, бизнес-задачам, тренд-вотчингу, веб-промо и первому стартапу. Интерфейс читает записи из `data/diary-entries.js`; открой `index.html` в браузере.

## Возможности

- Просмотр всех заметок с датой и темой
- Фильтр по области фокуса (цели обучения)
- Поиск по тексту заметки
- Структура полей заточена под новичка: вывод, следующие шаги, источники
- Визуальная система в `design.json` (редакционный «бумажный» стиль) — ориентир для вёрстки и для ИИ; реализация в `css/styles.css`

## Правила внесения заметок

### Обязательно

1. **Тема (`topic`)** — одна короткая строка. Сразу видно, о чём заметка (например: «Первый промт для Cursor», «Что такое MCP»).
2. **Дата (`date`)** — день, **когда сформирована мысль или закончена сессия**, в формате `YYYY-MM-DD`. Не дата «когда-нибудь переписать», а когда информация актуальна для тебя; так позже видно, не устарело ли.
3. **Области (`areas`)** — хотя бы один ключ из `data/areas.js`: обучение, автоматизации, маркетинг, бизнес, стартап, тренд-вотчинг, веб/промо, инструменты.
4. **Содержание (`content`)** — основной текст; абзацы разделяй пустой строкой (`\n\n`).

### Сильно рекомендуется

5. **`summary`** — одна строка: о чём заметка, если темы мало.
6. **`insight`** — один главный вывод («что я теперь понимаю / что работает»).
7. **`nextSteps`** — список конкретных действий до следующей сессии.
8. **`toolsMentioned`** — Cursor, MCP, конкретные сервисы — чтобы потом искать по стеку.
9. **`sources`** — ссылки на доки, статьи, видео: `{ title, url }`.

### Порядок в файле

- Новые записи добавляй **в начало массива** `DIARY_ENTRIES` в `data/diary-entries.js` — в интерфейсе так удобнее листать от свежих к старым.

### Как вносить

- Редактируй `data/diary-entries.js` вручную или попроси ассистента в Cursor добавить запись по этим правилам (в репозитории есть правило `.cursor/rules/vibe-coding-diary.mdc`).

### Отдельные темы

- Про **текущий бизнес** для рекламы и веб-интерфейсов — заведи одну или несколько заметок с темой вроде «Контекст бизнеса для промтов» и храни там УТП, аудиторию, ограничения — чтобы в других чатах не повторять всё с нуля.
- Про **плагины Cursor и MCP** (например Miro): команды в чате, что умеет сервер, как залогиниться — удобно фиксировать отдельной заметкой в `diary-entries.js`, чтобы не искать объяснение заново.
- Про **структуру контекста бизнес + личное** — см. заметку `2026-03-29-kontekst-hub` в `data/diary-entries.js` и папку `projects/biznes-i-lichnyy-kontekst/`.
